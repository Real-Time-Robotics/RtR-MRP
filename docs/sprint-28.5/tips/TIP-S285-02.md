## CONTEXT

Home dashboard hiện hiển thị OEE 0.0% · Uptime 0.0% · Quality Rate 0.0% với background đỏ — cảnh báo giả. User mới nghĩ app hỏng. Thực tế DB chưa có data.

## TASK

### 1) Detect empty state

- Trước khi render KPI card, query: có WO/Plan/Equipment/ShiftReport nào trong DB không?
- Tạo helper `hasProductionData()` server-side fetch count.

### 2) Replace empty KPI card

Khi `hasProductionData() === false`:
- Thay 9 KPI card → 1 hero card lớn:
  - Icon Factory + heading "Chào mừng đến RTR-MRP"
  - Mô tả: "Tạo work center + work order đầu tiên để bắt đầu theo dõi sản xuất."
  - 3 button CTA:
    - "Tạo Work Center" → `/dashboard/production/work-centers/new` (nếu route có).
    - "Tạo Work Order" → `/dashboard/production/new` (verify).
    - "Tạo Daily Plan" → `/dashboard/production/daily-plan?action=create`.
  - Dưới: link "Xem tour 3 phút" (TIP-S285-05 sẽ wire).

### 3) Khi có data

Render 9 KPI card như cũ, nhưng:
- OEE/Uptime/Quality Rate: nếu value = 0 + denominator = 0 (chưa có scheduled time hay run time) → hiển thị "—" thay 0.0%, background xám không đỏ.
- Chỉ background đỏ khi có data thật + value < threshold (vd OEE < 60% mới đỏ).

### 4) Helper

`src/lib/dashboard/empty-state.ts`:
```ts
export async function hasProductionData(): Promise<boolean> {
  const [wo, plan, eq] = await Promise.all([
    prisma.workOrder.count({ where: { status: { in: ['IN_PROGRESS', 'COMPLETED'] } } }),
    prisma.dailyProductionPlan.count(),
    prisma.equipment.count(),
  ]);
  return wo > 0 || plan > 0 || eq > 5;  // threshold tránh count seed admin user
}
```
