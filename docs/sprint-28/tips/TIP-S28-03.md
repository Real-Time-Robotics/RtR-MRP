## CONTEXT

Schema `DailyProductionPlan` + `DailyProductionPlanLine` đã có (TIP-01). Trang `/dashboard/production/daily-plan` hiện skeleton (TIP-02). TIP này fill thật.

**User chính:** quản đốc (role `production` hoặc `admin`).

**Workflow nghiệp vụ:**
1. Đầu ngày/tuần: quản đốc chọn ngày + work center → tạo Plan DRAFT.
2. Add WO line vào plan (chọn từ list WO chưa assigned cho ngày đó).
3. Assign operator + estimated time per line.
4. Approve plan → status APPROVED.
5. Đến ngày, supervisor/operator nhìn plan → bắt đầu thực thi.
6. Cuối ngày: status COMPLETED auto khi mọi line WO đã DONE.

## TASK

### 1) API routes

**`POST /api/production/daily-plan`** role `production|admin`
- Body Zod: `{ date: ISO date, workCenterId: cuid, notes?: string }`
- Behavior: tạo plan DRAFT, gen `planNumber` format `DPP-YYYYMMDD-<NN>` (NN = sequential per ngày).
- 201 `{ plan }`.
- Conflict: nếu đã có plan cùng `(date, workCenterId)` → 409 với existing planId.

**`GET /api/production/daily-plan?date=&workCenterId=`** role authenticated
- Query plan + lines + WO + operator info.
- Hỗ trợ range: `?from=&to=` cho weekly view.
- 200 `{ plans: [...] }`.

**`POST /api/production/daily-plan/:id/lines`** role `production|admin`
- Body: `{ workOrderId, sequence?, plannedQty, assignedToUserId?, estimatedStartTime?, estimatedEndTime?, notes? }`
- Validate WO chưa được add vào plan khác cùng ngày.
- 201 `{ line }`.

**`PUT /api/production/daily-plan/:id/lines/:lineId`** role `production|admin`
- Update line (qty, operator, time).

**`DELETE /api/production/daily-plan/:id/lines/:lineId`** role `production|admin`

**`PUT /api/production/daily-plan/:id/lines/reorder`** role `production|admin`
- Body: `{ lineIds: string[] }` array order mới.
- Update sequence theo array index.
- 200 `{ plan }`.

**`POST /api/production/daily-plan/:id/approve`** role `production|admin`
- Set status `APPROVED`, approvedByUserId, approvedAt.
- Validate có ≥ 1 line.

**`POST /api/production/daily-plan/:id/cancel`** role `production|admin`
- Set status `CANCELLED`.

**`GET /api/production/daily-plan/late-wos?date=`** role authenticated
- Trả list WO trễ (dueDate < date && status != COMPLETED && status != CANCELLED).
- Suggest re-plan: với mỗi WO trễ, tìm plan tiếp theo cùng work center.

### 2) UI `/dashboard/production/daily-plan/page.tsx`

**Layout 2 cột:**

**Cột trái (3/4):** Weekly grid (7 cột Mon-Sun)
- Mỗi cột = 1 ngày. Header: ngày + work center selector (dropdown).
- Trong cột: list line của plan ngày đó (card draggable). Card: WO number · qty · operator · time.
- Click card → open dialog edit (assign operator, time).
- Drag card sang ngày khác → move line. Drag reorder trong cùng ngày → update sequence.

**Cột phải (1/4):** Sidebar phụ
- "WO chưa assigned" list (filter bởi work center selected).
- Drag từ sidebar phụ → drop vào cột → tạo line mới.
- "WO trễ ⚠️" section: list WO dueDate < today, status != COMPLETED. Click → button "Dời sang ngày kế tiếp" tạo line mới ngày + 1.

**Header trang:**
- Date range picker (default tuần hiện tại).
- Work center filter (multi-select).
- Button "Approve hôm nay" (nếu plan today DRAFT).

**Indicator:**
- Plan DRAFT: viền dashed.
- Plan APPROVED: viền solid xanh.
- Line WO COMPLETED: text strikethrough + check icon.
- Plan trễ (dueDate < today, status != COMPLETED): viền đỏ.
