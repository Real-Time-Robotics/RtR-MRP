## CONTEXT

Operator dùng tablet/phone trên shop floor. UI phải mobile-first (375px), thao tác nhanh, ít chữ. Login → bắt đầu ca → list WO assigned → click WO → nhập qty produced/scrap → submit. Cuối ca → end shift.

**Schema dùng:** `Shift`, `ShiftAssignment`, `LaborEntry`, `WorkOrder`, `BomHeader`, `BomLine`, `Part`.

## TASK

### 1) API routes

**`POST /api/production/shift-entry/start`** role `production|admin`
- Body: `{ shiftId, workCenterId, date }`
- Behavior: 
  - Tạo `ShiftAssignment(employeeId=user, shiftId, workCenterId, date, status=checked_in, actualStart=now)`.
  - Tạo `LaborEntry(userId, workCenterId, type=DIRECT, startTime=now, endTime=null)` để track ca chính.
  - Validate user chưa active ca khác cùng date.
- 201 `{ shiftAssignment, mainLaborEntry }`.

**`GET /api/production/shift-entry/active`** role authenticated
- Trả ShiftAssignment đang active của user (status=checked_in, actualEnd=null).
- 200 `{ active: ShiftAssignment | null, assignedWOs: WorkOrder[] }` — WO list lấy từ DailyProductionPlan.lines của ngày + workCenter.

**`POST /api/production/shift-entry/append-qty`** role `production|admin`
- Body: `{ workOrderId, quantityProduced, quantityScrapped, notes? }`
- Behavior:
  - Validate user có active ShiftAssignment.
  - Tạo `LaborEntry(userId, workOrderOperationId=null, workCenterId, type=DIRECT, startTime, endTime, durationMinutes, quantityProduced, quantityScrapped)`.
  - Update `WorkOrder.completedQty += quantityProduced`, `WorkOrder.scrapQty += quantityScrapped`.
  - Nếu `WorkOrder.completedQty >= WorkOrder.quantity` → tự update WO.status = COMPLETED + trigger generateSerialsForCompletedWO (Sprint 27 đã có).
- 200 `{ laborEntry, workOrderUpdated }`.

**`POST /api/production/shift-entry/end`** role `production|admin`
- Body: `{ notes? }`
- Behavior:
  - Tìm active ShiftAssignment + main LaborEntry.
  - Set ShiftAssignment.actualEnd = now, status = checked_out, durationMinutes calc.
  - Set main LaborEntry.endTime = now, durationMinutes calc.
  - Optional: trigger ShiftReport.generate (TIP-06 sẽ implement, có thể stub call).
- 200 `{ shiftAssignment, summary: { totalProduced, totalScrap, durationHours } }`.

**`GET /api/production/shift-entry/wo/:id/bom`** role authenticated
- Trả BOM của WO + qty required vs qty issued (từ MaterialAllocation).
- 200 `{ bomLines: [{partNumber, name, qtyRequired, qtyIssued}] }`.

### 2) UI `/dashboard/production/shift-entry/page.tsx`

**Mobile-first, viewport 375px primary.**

**State 1 — Chưa bắt đầu ca:**
- Header: avatar + tên user.
- Card lớn: "Chưa bắt đầu ca".
- Form:
  - Select Work Center (dropdown).
  - Select Shift (3 ca cố định: Sáng/Chiều/Đêm — đọc từ Shift table).
  - Button lớn "Bắt đầu ca" (primary, full-width, ≥ 56px tall).

**State 2 — Đang trong ca:**
- Header: avatar + tên user + badge ca + clock running ("00:23:45").
- Card "Ca hiện tại": Work center · Shift · Đã làm: N qty · Scrap: M.
- List WO assigned (từ DailyProductionPlan):
  - Mỗi WO 1 card lớn (touch target ≥ 80px tall):
    - WO number + product name.
    - Progress bar: completedQty / targetQty.
    - Button "Nhập qty" (primary).
    - Icon xem BOM (collapse/expand inline).
  - Click "Nhập qty" → modal full-screen mobile:
    - Input number "Qty produced" (numeric keyboard, default 0).
    - Input number "Qty scrap" (default 0).
    - Textarea ngắn "Ghi chú" (optional).
    - Button "Lưu" + "Huỷ".
- Floating button bottom-right "Báo sự cố" (đỏ, mở modal TIP-05 downtime quick).
- Floating button bottom-left "Kết thúc ca" (warning màu).

**State 3 — Kết thúc ca:**
- Modal summary: tổng produced, scrap, downtime, hours.
- Confirm button → end shift API call → redirect về State 1.

**Touch UX:**
- Button ≥ 44×44 px.
- Font size ≥ 16px (input avoid iOS auto-zoom).
- Spacing thoáng (12-16px gap).
- No hover-only interaction.
