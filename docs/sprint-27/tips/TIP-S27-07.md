**File:** `src/app/(dashboard)/operations/work-order/page.tsx`

**Layout:**
- Header: "Gia công" + count "N WO đang mở".
- Filter row: status (DRAFT, IN_PROGRESS, COMPLETED, CANCELLED) + search by WO number + date range (from-to).
- Table columns: WO Number · Product · Target Qty · Completed Qty · Status (badge) · Assigned · Created · Actions.
- Action button per row:
  - Status DRAFT/IN_PROGRESS → button "Hoàn thành" (mở dialog confirm).
  - Status COMPLETED → button "Xem serial sinh ra" (link `/dashboard/search/serial?productionReceipt=<id>` hoặc tương tự).
- Dialog "Hoàn thành":
  - Input completedQty (default = targetQty).
  - Input notes (optional).
  - Button "Xác nhận" gọi `POST /api/work-orders/:id/complete` (hoặc route đã grep tìm).
  - Sau success: toast "Đã hoàn thành WO. N serial sinh ra." (nếu API trả `serialsGenerated`).
  - Refresh table data.

**Data fetching:**
- Server component fetch initial list, hoặc client với SWR/React Query.
- Pagination: 25/page.

**Permission:** server check role `production`, `engineer`, `admin` qua middleware nếu route đã có. Nếu không có middleware tự thêm `requireRole` ở API call (TIP-03 đã wrap).

### 2) `/dashboard/operations/assembly/page.tsx` — Lắp ráp

**File:** `src/app/(dashboard)/operations/assembly/page.tsx` (list view)
**File:** `src/app/(dashboard)/operations/assembly/[id]/page.tsx` (detail/scan view)

**List view:**
- Header: "Lắp ráp" + "Tạo AO mới" button.
- Filter: status, productId, assignedToUserId.
- Table: AO Number · Product · Target Qty · Status · Assigned · Created · Actions.
- Action: 
  - DRAFT → "Bắt đầu" (start).
  - IN_PROGRESS → "Quét serial" (link sang detail).
  - COMPLETED → "Xem serial cha" (link `/dashboard/search/serial/<parent>`).

**Create AO dialog:**
- Select product (autocomplete combobox).
- Select bomHeader (default = active mới nhất; show version).
- Input targetQuantity (default 1).
- Select assignedToUserId (optional).
- Submit `POST /api/assembly` → redirect đến detail.

**Detail view (`[id]/page.tsx`):**
- Header: AO number + status badge + assigned user.
- Section "BOM Required" (read-only): table line số · component name · qty required.
- Section "Scan child serial":
  - Input text "Quét hoặc nhập serial" (autofocus, hỗ trợ Enter submit).
  - Sau submit: gọi `POST /api/assembly/:id/scan-child` body `{childSerial}`.
  - Validate response:
    - 200 → toast success + cập nhật progress.
    - 400 reason='NOT_FOUND' → toast "Không tìm thấy serial".
    - 400 reason='NOT_IN_STOCK' → toast "Serial không ở trạng thái IN_STOCK".
    - 400 reason='NOT_IN_BOM' → toast "Serial không thuộc BOM".
    - 400 reason='ALREADY_USED' → toast "Serial đã dùng ở AO khác".
- Section "Progress":
  - Render progress bar `byBomLine[].scanned / .required`.
  - Mỗi line có button "Bỏ quét" (call unscan) hoặc list serial đã scan (collapse).
- Footer:
  - Button "Hoàn thành lắp ráp" (disabled nếu chưa scan đủ): gọi `POST /api/assembly/:id/complete`.
  - Button "Huỷ AO": gọi `POST /api/assembly/:id/cancel` (confirm dialog).
- Sau complete success:
  - Show modal "EBOX serial sinh ra: <parentSerial>" với link `/dashboard/search/serial/<parentSerial>`.
  - Update status badge.

### 3) `/dashboard/operations/issue/page.tsx` — Xuất hàng

**File:** `src/app/(dashboard)/operations/issue/page.tsx`

**Layout:**
- Header: "Xuất hàng" + count "Hôm nay xuất N đơn vị".
- 2 mode tabs hoặc radio:
  - **Xuất chính** (XUẤT_CHÍNH): chọn 1 EBOX serial (parent) — sẽ link cùng children theo SerialLink.
  - **Xuất lẻ**  (XUẤT_LẺ):  chọn 1 SerialUnit bất kỳ.
- Form fields:
  - Input "Quét hoặc nhập serial" (autofocus).
  - Lookup hiển thị: product, status, source, location (gọi `GET /api/serial/:serial`).
  - Validation client-side: chỉ cho phép status `IN_STOCK` hoặc `ALLOCATED`.
  - Input customer/destination (optional, free text).
  - Input notes (optional).
- Button "Xác nhận xuất":
  - Gọi `POST /api/serial/:serial/status` body `{status: 'SHIPPED', note: '<destination> | <notes> | XUẤT_CHÍNH/LẺ'}`.
  - Nếu mode XUẤT_CHÍNH và serial có children:
    - Gọi cùng API cho mỗi child serial để chuyển sang SHIPPED.
    - Hiện list children được xuất kèm.
  - Refresh, clear form, ready cho serial tiếp theo.
- Lịch sử xuất: table 10 dòng gần nhất (gọi `GET /api/serial?status=SHIPPED&limit=10` hoặc query lot transactions). Optional, có thể skeleton.
