### 3) Service — `src/lib/assembly/index.ts` (hoặc `service.ts`)

**Helpers cần export:**

```ts
/** Generate next AO number, format AO-YYYY-NNNN. Counter increments per year. */
export async function generateAoNumber(now?: Date): Promise<string>;

/** Validate child serial against AO's BOM. Returns reason string or null if OK. */
export async function validateChildSerial(
  aoId: string,
  childSerial: string
): Promise<{ ok: true; childUnit: SerialUnit; bomLine: BomLine } | { ok: false; reason: string }>;

/** Complete an AO. Returns parentSerial string + AO updated. */
export async function completeAssemblyOrder(
  aoId: string,
  userId: string
): Promise<{ parentSerial: string; aoId: string }>;
```

**`generateAoNumber()` logic:**
- Format `AO-{YYYY}-{NNNN}` zero-pad 4 digit.
- Counter từ DB: `SELECT count(*) FROM assembly_orders WHERE EXTRACT(YEAR FROM "createdAt") = currentYear` rồi +1.
- Race-safe: đặt trong transaction tạo AO; nếu unique constraint fail → retry tối đa 3 lần với +1 counter. Đơn giản, không cần per-counter table.

**`validateChildSerial()` rules:**
1. Lookup `SerialUnit` by `serial` (unique). Nếu không có → `{ ok: false, reason: 'NOT_FOUND' }`.
2. Nếu `status !== IN_STOCK` → `{ ok: false, reason: 'NOT_IN_STOCK' }`.
3. Lookup AO's `bomHeader.lines`. Tìm `bomLine` có `partId` hoặc moduleDesignId match với child unit's product/moduleDesign. Nếu không có line nào match → `{ ok: false, reason: 'NOT_IN_BOM' }`.
4. Check `SerialLink.findUnique({ where: { childSerialId: childUnit.id } })`. Nếu đã có → `{ ok: false, reason: 'ALREADY_USED' }`.
5. Else → `{ ok: true, childUnit, bomLine }`.

**`completeAssemblyOrder()` logic (tất cả trong `prisma.$transaction`):**
1. Lookup AO + bomHeader.lines (include count).
2. Lookup tất cả SerialLink hiện có với `assemblyOrderId = aoId` → đếm theo bomLineId.
3. Validate: mỗi `bomLine` (theo `quantity`) phải có đủ số lượng child link đã quét. Nếu thiếu → throw `IncompleteAssemblyError(bomLineId, expected, actual)`.
4. Lookup `bomHeader.product.moduleDesign` (qua field reference — nếu Product không có ModuleDesign trực tiếp, tìm `ModuleDesign.bomHeaderId === bomHeader.id`). Nếu vẫn không có → throw `NoModuleDesignForProductError`.
5. Call `generateSerial({ moduleDesignId, prismaClient: tx })`.
6. Tạo `SerialUnit` parent với:
   - `serial = <generated>`
   - `productId = ao.productId`
   - `moduleDesignId = <found>`
   - `status = IN_STOCK`
   - `source = MANUFACTURED`
   - `createdByUserId = userId`
7. Update tất cả child `SerialUnit` → `status = CONSUMED`.
8. Update AO: `status=COMPLETED`, `parentSerialId=<new parent id>`, `completedAt=now`.
9. Return `{ parentSerial: serial, aoId }`.

### 4) API routes

Tất cả wrap `requireRole(...)` + Zod + try/catch.

**`POST /api/assembly`** — role: `production`, `engineer`, `admin`
- Body: `{ productId: string.cuid(), bomHeaderId?: string.cuid(), targetQuantity?: number(default 1), assignedToUserId?: string.cuid(), notes?: string }`
- Behavior: 
  - Nếu `bomHeaderId` không cung cấp → tự pick BomHeader active mới nhất của product.
  - Generate `aoNumber`.
  - Tạo AO status=DRAFT.
  - Return 201 `{ ao: {...} }`.

**`POST /api/assembly/:id/start`** — role: `production`, `admin`
- Set status=IN_PROGRESS, startedAt=now. Idempotent (nếu đã IN_PROGRESS → 200 noop).
- Validate: chỉ cho phép DRAFT → IN_PROGRESS.

**`POST /api/assembly/:id/scan-child`** — role: `production`, `admin`
- Body: `{ childSerial: string }`
- Behavior:
  - Validate AO status = IN_PROGRESS (không cho scan ở DRAFT/COMPLETED/CANCELLED).
  - Call `validateChildSerial(aoId, childSerial)`.
  - Nếu `ok=false` → 400 với `reason` body.
  - Nếu `ok=true` → tạo `SerialLink(parentSerialId=null until complete, childSerialId, bomLineId, assemblyOrderId, linkedByUserId)`.
  - **Lưu ý:** parent chưa có lúc scan, để parent null không hợp lệ vì SerialLink.parentSerialId NOT NULL. → **Refactor:** đặt SerialLink chỉ tạo lúc COMPLETE, còn lúc scan thì lưu vào bảng tạm hoặc cờ trên SerialUnit (`SerialUnit.allocatedToAoId`).
  - **Quyết định:** dùng cách đơn giản — set `SerialUnit.status = ALLOCATED` + thêm field `meta.allocatedToAoId = aoId` (JSON meta đã có trong TIP-02). Không tạo SerialLink ngay. SerialLink sẽ tạo batch ở step COMPLETE.
  - Return 200 `{ scannedCount, totalRequired, byBomLine: [{bomLineId, scanned, required}] }`.

**`POST /api/assembly/:id/unscan`** — role: `production`, `admin`
- Body: `{ childSerial: string }`
- Behavior: revert child unit back IN_STOCK + clear `meta.allocatedToAoId`. Idempotent.

**`POST /api/assembly/:id/complete`** — role: `production`, `admin`
- Behavior: call `completeAssemblyOrder(aoId, userId)`. Errors:
  - `IncompleteAssemblyError` → 422 `{ error, missingByBomLine }`.
  - `NoModuleDesignForProductError` → 422.
  - `SerialNumberingRuleNotFoundError` → 422.
- Return 200 `{ parentSerial, aoId, status: COMPLETED }`.
- Side effect (in transaction): update children status to CONSUMED + create N SerialLink + create parent SerialUnit + update AO.

**`POST /api/assembly/:id/cancel`** — role: `production`, `admin`
- Behavior: AO status=CANCELLED, cancelledAt=now. Children status revert IN_STOCK + clear meta.allocatedToAoId.

**`GET /api/assembly/:id`** — role: tất cả authenticated
- Trả AO + bomHeader.lines + scannedCount per line + parentSerial (nếu COMPLETED).

**`GET /api/assembly`** — role: tất cả authenticated  
- Query: `?status=&productId=&assignedToUserId=&page=&limit=`. Pagination.

### 5) WO → Serial wiring

**Mục tiêu:** khi `WorkOrder.completedQty` tăng và Product/Module có `serialControl=true`, tự động sinh `SerialUnit`.

**Cách triển khai:**
1. **Tìm route hiện tại** complete WO. Có thể là `POST /api/manufacturing/:id/complete` hoặc `PATCH /api/work-orders/:id` body `{status: 'COMPLETED'}`. Grep tìm trước. Nếu không có → thêm endpoint mới `POST /api/work-orders/:id/complete`.
2. Trong handler, sau khi update WO + tạo ProductionReceipt (giữ nguyên logic hiện có):
   - Lookup `Product.serialControl` hoặc tìm `ModuleDesign` link với productId. Nếu không phát hiện cần serial → return như cũ (non-regression).
   - Loop `completedQty` lần (hoặc `delta = newCompletedQty - prevCompletedQty`):
     - Call `generateSerial({ moduleDesignId, prismaClient: tx })`.
     - Tạo `SerialUnit` với `source=MANUFACTURED`, `status=IN_STOCK`, `productId`, `moduleDesignId`, `productionReceiptId=<receipt id>`, `createdByUserId=session.user.id`, `inventoryId=<receipt's inventoryId nếu có>`.
3. Nếu generateSerial fail (no rule) → log warning + skip serial generation (không block WO complete). Ghi vào `meta` của ProductionReceipt: `serialGenerationSkipped: true, reason: 'NO_RULE'`.
4. Nếu generate fail nửa chừng (5/10 sinh được, 6 fail) → rollback toàn bộ transaction. WO complete bị rollback. Trả 500 `{error, partialFailure: true}`.

**File đụng:**
- Locate qua grep `manufacturing.*complete` hoặc `WorkOrder` handler.
- Nếu file chứa nhiều logic khác (BOM consumption, lot transactions) → KHÔNG đụng phần đó. Chỉ append serial generation cuối transaction.
