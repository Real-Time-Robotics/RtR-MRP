## TASK

### 1) Script vị trí: `scripts/migrate-dashboard-xlsx.ts`

CLI entry, chạy qua `npx tsx scripts/migrate-dashboard-xlsx.ts [options]` hoặc qua `npm run migrate:dashboard`. Thêm npm script vào `package.json`:
```json
"migrate:dashboard": "tsx scripts/migrate-dashboard-xlsx.ts"
```

**CLI flags:**
- `--file <path>` (default: `./data/DASHBOARD-da1f05cc.xlsx`)
- `--dry-run` — không commit DB, chỉ in count.
- `--only <entity>` — (optional) chỉ import 1 entity, vd `--only parts`. Dùng khi debug.
- `--skip <entity,entity>` — bỏ qua entity.
- `--mapping-file <path>` (default: `./scripts/dashboard-category-mapping.json`) — mapping 138 category → cluster.

### 3) Category clustering (REQ-27-09 + risk #1 trong Blueprint)

**Workflow:**
1. Script chạy pre-flight `dry-run --mode=cluster-preview`:
   - Đọc 138 category string unique.
   - Suggest cluster rule theo keyword matching (vd mọi string chứa "resistor" → cluster `ELEC_PASSIVE_RESISTOR`; chứa "cap" + "lipo|lion" → `BATTERY`).
   - Xuất ra file `./scripts/dashboard-category-mapping.json`:
     ```json

         { "code": "MCU",                    "name": "Vi điều khiển", "parent": "ELECTRONICS" },

         ...
       ],
       "mapping": {
         "Điện trở 10k 0603 SMD": "ELEC_PASSIVE_RESISTOR",
     ```

2. Chủ thầu review file này trước khi chạy import thật. File có thể edit tay.
3. Script import thật đọc file này, KHÔNG tự cluster lại.
4. Nếu file có entry trong `unresolved` → script refuse chạy (exit 1, in ra danh sách chưa có mapping).

**Rule cluster gợi ý (tự do điều chỉnh, ghi lý do vào DEVIATIONS):**
- ~20-30 cluster cuối cùng, 2 tier (parent `ELECTRONICS` / `MECHANICAL` / `PACKAGING` / `CONSUMABLE`, rồi child cluster).

### 4) Mapping value → enum

- `SerialUnit.status`:
  - `"TỒN"` → `IN_STOCK`
  - `"ĐÃ XUẤT"` / `"XUẤT"` → `SHIPPED`
  - `"ĐÃ DÙNG"` / `"CONSUMED"` → `CONSUMED`
  - `"LỖI"` / `"HỎNG"` → `SCRAPPED`
  - `"TRẢ"` / `"RETURN"` → `RETURNED`
  - Default unknown → `IN_STOCK` + log warning.
- `SerialUnit.source`:
  - `"GIA CÔNG"` / `"GIA_CONG"` → `MANUFACTURED`
  - `"NHẬP"` / `"MUA"` → `RECEIVED`
  - Unknown → `IMPORTED` (vì đây là migration từ Excel → đúng nghĩa).
- `BomLine.sourceType` từ BOM_CHUAN col `TYPE`:
  - `"INTERNAL"` / `"GIA CÔNG"` → `INTERNAL`
  - `"EXTERNAL"` / `"MUA"` → `EXTERNAL`
  - Default → `INTERNAL`.
- `LotTransaction.type`:
  - NHAP_KHO sheet → `RECEIVED`
  - XUAT_KHO sheet → `ISSUED`
- User match (`SerialUnit.createdByUserId`): field `USER` trong SERIAL_MASTER là short name (vd `HAI`, `hải`, `LÂM`). Script:
  - Build dict User theo `name` lowercase match một phần.
  - Nếu không match → `createdByUserId = null` + log warning.

### 5) Idempotency

- Mỗi entity upsert by unique key (xem bảng trên). Chạy lại script lần 2:
  - Category: count không tăng.
  - Part: count không tăng, field update nếu input đổi.
  - BomLine: delete-then-recreate per BomHeader (BomHeader code unchanged, xóa BomLine cũ rồi insert lại để không duplicate).
  - SerialUnit: upsert by `serial` (unique). Update field status/location theo sheet mới nhất.
  - LotTransaction: upsert by composite hash (deterministic) — nếu 1 row trùng date+part+qty+type → skip.
- In log: `X created / Y updated / Z skipped` cho mỗi entity.

### 6) Dry-run

- `--dry-run` chạy full parsing + validation nhưng **wrap trong `prisma.$transaction` và rollback ở cuối**.
- Output: bảng count expected vs actual (giống dưới đây).
- Nếu có mapping unresolved hoặc enum unknown → in warning + exit 2 (warning exit code khác thành công).

## CONSTRAINTS

- **KHÔNG** đụng schema — chỉ INSERT/UPDATE. Nếu phát hiện schema thiếu field → STOP + escalate Chủ thầu.
- **KHÔNG** xóa data ngoài scope (không `deleteMany` broad). Rollback duy nhất là transaction wrapper của dry-run.
- **KHÔNG** commit file Excel vào git. `data/*.xlsx` phải có trong `.gitignore` (kiểm tra trước, add nếu thiếu).
- **KHÔNG** hardcode path absolute của Lâm. Dùng `process.cwd()` + relative.
- **KHÔNG** import `Câu trả lời biểu mẫu 2` và `LAP_RAP_LOG` (scope explicit excluded).
- **KHÔNG** tự fill user mặc định cho SerialUnit không match. Để null + log warning.
- **Reuse** Prisma singleton client từ `src/lib/prisma.ts` (hoặc tương đương). Không tạo PrismaClient mới trong script (gây leak connection).
- **Package**: nếu chưa có, **ưu tiên `xlsx` (SheetJS)** thay vì `exceljs` vì dễ xử lý sheet có merged cell và Vietnamese diacritics. Check `package.json` trước khi install.
- **Idempotent** là P0. Nếu không đảm bảo idempotent → đừng commit. Hỏi Chủ thầu.
- **Non-TTY safe:** script chạy được trong CI (không prompt input). Mọi option qua CLI flag hoặc env var.
- Ambiguity nhỏ → tự quyết + ghi DEVIATIONS. Ambiguity lớn (mapping 5 category không rõ) → liệt kê vào `unresolved` và escalate.
